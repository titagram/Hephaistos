Perché non può deadlockare

  • Nessun implementer crea worktree, integra o attende l'altro.
  • Le quattro ownership di file sono disgiunte.
  • Gli stati avanzano soltanto in avanti: nessun lock reciproco.
  • Un'escalation congela una lane, non cancella né blocca l'altra.
  • Next.js dipende dall'evento EXPRESS_INTEGRATED, non dal processo Express.

Condizioni di arresto

  • Anchor o digest diversi dal manifest.
  • Necessità di modificare file condivisi o registry.
  • Review non chiusa dopo un repair.
  • Gate post-cherry-pick rosso: stop ed escalation, mai attesa indefinita.
